Key Findings

  • Project dashboards remain secondary in most coding tools.
  • Browser automation is usually external instead of embedded.
  • Workspace state is rarely promoted to a first-class artifact.
  • Users want low-friction context switching without losing agent state.

Decision Frame

Keep dashboards local-first Lean into multi-surface workflows Avoid chat-first product framing

Best positioning is as a cockpit for project execution, not as another assistant shell with a few side panels bolted on.
